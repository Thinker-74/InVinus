# Sessione 7 — 2026-03-31 — Security fix RLS candidature

## Cosa è stato fatto

Fix alert Supabase Security Advisor: la policy `insert_public` sulla tabella `candidature`
aveva `WITH CHECK (true)`, permettendo a chiunque (utenti anonimi) di inserire candidature
con `stato` arbitrario o con `note_admin` precompilate.

### Soluzione applicata
```sql
ALTER POLICY insert_public ON public.candidature
  WITH CHECK (stato = 'in_attesa' AND note_admin IS NULL);
```

### File modificati
- `db/migrations/001_security_fixes.sql` — aggiunto il fix con commento esplicativo

### Impatto
- Nessun impatto sul flusso normale (il form pubblico invia sempre stato default + note_admin NULL)
- Chiude il vettore di attacco: bypass workflow admin + injection in note_admin
